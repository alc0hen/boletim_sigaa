Políticas de Privacidade e Transparência

Última atualização: 2025

A transparência é o pilar fundamental do Boletim App. Abaixo, detalhamos tecnicamente como seus dados são coletados, processados e armazenados.

1. Arquitetura Proxy e Segurança

Nossa aplicação atua como uma interface intermediária (proxy) entre o seu dispositivo e o SIGAA. Quando você realiza login, o servidor da aplicação estabelece uma conexão segura (HTTPS/TLS) diretamente com os servidores da sua instituição (IFAL/UFAL).

  • SSR (Server-Side Rendering): O processamento das páginas HTML do SIGAA ocorre no nosso servidor para extrair apenas os dados necessários (notas, faltas).
  • Proteção contra SSRF: Implementamos validações rigorosas para garantir que o servidor se comunique apenas com os domínios oficiais das instituições, bloqueando requisições para destinos desconhecidos.
2. Credenciais e Senhas

A forma como tratamos sua senha depende de como você utiliza o aplicativo:

  • Login Padrão (Sessão Única): Suas credenciais são mantidas apenas na memória volátil (RAM) do servidor durante a sessão ativa. Assim que você faz logout ou fecha o navegador, esses dados são descartados. Nenhuma senha é salva em disco ou banco de dados.
  • Vincular Conta (Recurso Opcional): Se você optar por "Vincular Conta" para login automático, sua senha será armazenada no nosso banco de dados. No entanto, ela é criptografada utilizando a biblioteca Fernet (criptografia simétrica AES-128 com autenticação HMAC), garantindo que nem mesmo os administradores do sistema possam lê-la diretamente. A chave de criptografia é mantida separada do banco de dados (variável de ambiente).
3. Dados Acadêmicos e Cache

Para melhorar a performance e reduzir a sobrecarga no SIGAA, utilizamos um sistema de cache inteligente:

  • Histórico Escolar: Os dados do seu histórico (notas passadas, CR) são armazenados em formato JSON criptografado no banco de dados.
  • Criptografia em Repouso: Assim como as senhas, esses dados são ilegíveis sem a chave de criptografia exclusiva do servidor.
  • Expiração: O cache é atualizado automaticamente a cada 3 dias ou quando você solicita uma atualização manual.
4. Código Aberto e Auditoria

Não pedimos que você confie cegamente em nós. O código-fonte completo da aplicação está disponível para auditoria pública. Qualquer pessoa com conhecimento técnico pode verificar exatamente como os dados são manipulados.

5. Dados de Terceiros (Google)

Ao utilizar o login com Google, coletamos apenas seu nome, e-mail e foto de perfil para criar sua identificação na plataforma. Não temos acesso à sua senha do Google ou a outros dados da sua conta Google.
